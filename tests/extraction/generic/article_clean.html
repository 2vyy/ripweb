Tokio: The Async Runtime for Rust

Tokio is an asynchronous runtime for Rust that provides the building blocks needed for writing networking applications. It gives you an event-driven, non-blocking I/O platform for writing asynchronous TCP and UDP network applications.

The core of Tokio is a work-stealing scheduler that efficiently drives async tasks across a thread pool. When a task is blocked on I/O, it yields back to the scheduler so other work can proceed. Once the I/O completes, the task is rescheduled.

Key Components

Tokio ships with several subsystems: an async I/O layer built on epoll/kqueue/IOCP, a timer wheel for high-resolution timeouts, synchronization primitives like Mutex and Semaphore, and channels for passing data between tasks without sharing memory.

Related: See also async-std, smol, and glommio for alternative runtimes.

The #[tokio::main] macro expands into a runtime builder call and wraps your async main function, making it straightforward to get started without manual runtime construction.
